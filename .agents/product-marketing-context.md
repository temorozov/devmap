# Product Marketing Context

*Last updated: 2026-04-30*

## Product Overview
**One-liner:** Skill Tree turns a learning goal into an interactive game-like map of skills, levels, prerequisites, and progress.
**What it does:** Users organize a topic as connected skill nodes, read each skill description, track the current level, and see what branch to work on next. The app can draft a tree with AI, but the core value is using the tree as a visual learning interface.
**Product category:** Learning roadmap, skill tree planner, visual progress tracker.
**Product type:** SaaS-style web app.
**Business model:** Not defined in the current repo.

## Target Audience
**Target companies:** Individual learners and self-directed builders rather than a company-first product.
**Decision-makers:** The learner using the app.
**Primary use case:** Make a confusing field feel playable and manageable by turning it into connected skills.
**Jobs to be done:**
- Understand what to learn next.
- Track progress without a static checklist.
- Keep motivation visible while learning.
**Use cases:**
- Learning frontend, math, language, music, fitness, design, or any complex topic.
- Practicing on the built-in demo tree before using a personal tree.

## Problems & Pain Points
**Core problem:** Roadmaps and long lists often explain what exists, but not what the learner should do next or how far they have progressed.
**Why alternatives fall short:**
- Static PDFs and checklists do not feel interactive.
- Notes apps hide dependencies and progress.
- Generic roadmaps can feel too large on day one.
**What it costs them:** Lost focus, unclear priorities, and abandoned learning plans.
**Emotional tension:** The user wants progress, but the field feels too broad and unstructured.

## Differentiation
**Key differentiators:**
- Game-like skill nodes with levels and max levels.
- Right-click skill details with title, description, icon, level, and max level.
- Click to level up, Shift-click to level down, and visible branch mastery.
- Optional AI draft for the first tree.
**How we do it differently:** The product makes learning feel like navigating a skill tree from a game instead of reading a linear roadmap.
**Why that's better:** The user can see what is unlocked, what is in progress, and what to improve next.
**Why customers choose us:** It gives a clear visual interface for practicing, leveling, and staying oriented.

## Customer Language

**How they describe us:**

**Words to use:** skill tree, skill, node, branch, level, max level, description, icon, progress, roadmap, unlock, right-click, Shift-click.
**Words to avoid:** productivity platform, learning management system, enterprise training suite.
**Glossary:**
| Term | Meaning |
|------|---------|
